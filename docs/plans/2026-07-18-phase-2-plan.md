## Task 2: CLI `inject-public-key` + `fix-appcast` commands

Two pure-logic commands added to `src/cli.ts` dispatch (logic in a new `src/appcast.ts` so cli.ts stays thin):

- `inject-public-key --file <path> --key <value> [--placeholder <string>]` â€” placeholder defaults to the `SPARKLE_ED_PUBLIC_KEY_PLACEHOLDER` constant already exported by `src/builder.ts` (import it â€” single source). Reads the file, errors (exit non-zero, message naming file and placeholder) if the placeholder does not occur â€” this mirrors Kansoku CI's fail-fast grep, the guard against silently shipping a build with no key. Replaces ALL occurrences, writes back. `--key` may also be sourced from env `SPARKLE_ED_PUBLIC_KEY` when the flag is omitted (error if neither).
- `fix-appcast <appcast.xml> --repo <owner/repo> [--tag-prefix <string>]` â€” port of `fix-appcast-enclosure-urls.mjs`, generalized: tag prefix defaults to `"v"` (Kansoku passes `desktop-v`). Same rewrite semantics: for each enclosure URL under `releases/download/<tagPrefix><anything>/<file>`, extract `\d+\.\d+\.\d+` from the filename and repoint to `releases/download/<tagPrefix><version>/<file>`; count and log rewrites.

## Task 4: Composite GitHub Action

`action/action.yml` (composite), encapsulating the Sparkle steps of Kansoku's `desktop-release.yml`:

- Inputs: `tag` (required), `archive-dir` (required â€” the dir containing this release's zip + optional sidecar `.md` notes), `ed-private-key` (required, the secret VALUE), `repo` (default `${{ github.repository }}`), `tag-prefix` (default `v`), `download-url-prefix` (default derived: `https://github.com/<repo>/releases/download/<tag>/`), `full-release-notes-url` (default `https://github.com/<repo>/releases`), `sparkle-version` (default `2.9.4`), `sparkle-sha256` (default the pinned hash), `delta-bases` (default `2` â€” how many previous releases' zips to fetch), `fetch-delta-bases` (default `true`), `publish` (default `false` â€” when `true`, runs `gh release create` with dmg/zip/appcast/deltas like Kansoku; when `false`, stops after generating, leaving publishing to the caller), `dmg-path` (optional, only used when publish=true), `notes-file` (optional, passed to gh release create).
- Steps (bash, macOS runner assumed â€” document that):
  1. Fetch Sparkle tools: curl the pinned tarball, `shasum -a 256 -c`, extract `bin/` to a temp dir, add to `GITHUB_PATH` (verbatim from the workflow, parameterized).
  2. Optionally fetch delta bases: `gh release list` filtered by `<tag-prefix>` (the workflow's explicit-filter trick â€” copy its comment rationale into a `#` comment, this is CI code where the workflow file already carried it), download previous `appcast.xml` + last N zips into `archive-dir`. Uses `GH_TOKEN` env the caller must provide.
  3. Sign + generate: RAM disk (`hdiutil attach ram://`, `diskutil erasevolume`), write key, run `generate_appcast` with `--embed-release-notes --download-url-prefix ... --full-release-notes-url ...`, then scrub (`dd` random overwrite, `rm`, detach) â€” verbatim port of the workflow's step, parameterized.
  4. Repoint enclosures: run the package CLI from the consumer workspace â€” `npx electron-sparkle-updater fix-appcast <archive-dir>/appcast.xml --repo <repo> --tag-prefix <tag-prefix>` â€” documenting in the action README section that the package must be an installed dependency of the calling workspace.
  5. Optional publish step (gated on `publish` input): `gh release create <tag> --title --notes-file <file> <assets>` mirroring Kansoku's asset list (dmg if given, zip(s) from archive-dir, appcast.xml, *.delta).
- No unit tests possible in-repo; verification = `actionlint` if available (`brew list actionlint || true` â€” if not installed, skip and say so) plus careful self-review, plus `bash -n` on any extracted step scripts. Keep every step's shell strict (`set -euo pipefail`).
- Verify: `pnpm test` (unchanged suite still green). Commit.
